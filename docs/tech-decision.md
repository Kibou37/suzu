# Техническое решение по п.5 ТЗ

## Что говорит ТЗ

> Платформа: CMS (WordPress с Elementor Pro / 1С‑Битрикс) **или** кастомная разработка на React + Node.js.

ТЗ **не обязывает** выбрать один конкретный вариант — заказчик оставил три альтернативы на выбор.

## Рекомендация проекта

**Кастомная разработка: Next.js (React) + NestJS (Node.js) + MySQL**

| Критерий | WordPress / Битрикс | React + Node.js |
|----------|---------------------|-----------------|
| Конфигуратор с расчётом цены | Сложно, плагины/кастом | Нативно |
| Личный кабинет | Плагины, ограничения | Полный контроль |
| CRM + банковские API | Интеграции через костыли | REST/Webhook из коробки |
| AI-чатбот | Ограниченно | Гибко |
| Скорость ≤ 2 с / 1,5 с | Риск без глубокой оптимизации | SSG/ISR + CDN |
| SEO | Хорошо (WP) | Хорошо (Next.js SSR) |
| Срок 12 недель | Реально для простого сайта | Реально при команде 4–5 чел. |
| Поддержка 6 мес | Зависимость от плагинов | Прозрачный код |

## Принятый стек

| Слой | Технология |
|------|------------|
| Frontend | Next.js 14+, TypeScript, Tailwind CSS |
| Backend | NestJS, Prisma |
| БД | MySQL 8, Redis |
| CMS | Strapi (headless) |
| Auth | JWT / NextAuth |
| Docker | frontend, backend, mysql, redis (docker compose) |
